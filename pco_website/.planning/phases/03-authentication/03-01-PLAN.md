Create `components/auth/LoginForm.tsx` — a "use client" controlled form component:

**Structure inside ChromeCard:**
1. "PSI CHI OMEGA" — use `SectionTitle` component (Cormorant Garamond heading)
2. `Divider` component (chrome line + green dot)
3. Email `<input>` — type="email", required
4. Password `<input>` — type="password", required
5. `ChromeButton` variant="primary" type="submit" — disabled while isSubmitting
6. Admin note: `<p className="text-xs text-white/40 text-center mt-2">Accounts are created by admins. Contact your chapter officer to get access.</p>`

**Behaviors to implement:**
- `useState` for: email, password, isSubmitting
- `useAuth()` to get `user` and `login`
- `useRouter()` for navigation
- Already-authenticated redirect: `useEffect(() => { if (user) router.replace('/dashboard'); }, [user, router])`
- Submit handler `handleSubmit(e: React.FormEvent)`:
  - `e.preventDefault()`
  - `setIsSubmitting(true)`
  - Call `apiFetch<AuthTokens & { user: User }>("/v1/auth/login", { method: "POST", body: JSON.stringify({ email, password }) })`
  - On success: call `login({ access_token: data.access_token, refresh_token: data.refresh_token }, data.user)` then `router.push('/dashboard')`
  - On error: `toast.error("Invalid email or password.")`
  - Always: `setIsSubmitting(false)` in finally

**Input styling (discretion — use Tailwind v4 @theme tokens):**
```tsx
<input

  onChange={(e) => setEmail(e.target.value)}
```
